The security model — read this once, properly

This site is static and publicly reachable. The "login" is therefore not a guard, it is a key: lessons, homework, and reports sit in the repo AES-GCM-encrypted (key derived from your password with PBKDF2, 210 000 iterations). A wrong password opens nothing because it decrypts nothing — no password hash is stored, anywhere. The login screen is a deterrent; the encryption is the actual protection.

⚠ Lost password = content unrecoverable, permanently. There is no reset, no second key. Put the password in your password manager — now.

By default the password lives only in the memory of the open page, so every page asks again. The lock screen's "stay unlocked while this tab is open" option keeps it in sessionStorage instead — gone when the tab closes. Use the Lock button in the top bar to end it earlier.

GitHub connection (auto-commit your reports)

A fine-grained PAT, scoped to this repo only, with just "Contents: Read and write". The token stays exclusively in this browser's localStorage and never enters the repo. Everything works without it too — after homework you then get a "download report" button for a manual commit.

Change password

Re-encrypts the login canary. Existing content stays under the old password until you run "Re-encrypt everything" below — until then you still need the old password to read old lessons.

Re-encrypt everything

Fetches every .enc file from the repo, decrypts it with the old password, encrypts it with the new one, and commits it back. Needs token + repo (above). Afterwards only the new password works, everywhere.
